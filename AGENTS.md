# Repository instructions

Read `CLAUDE.md` completely before changing this repository. Follow its locked invariants, verification requirements, isolated frontend/backend deploy-boundary rules, and small scoped commit conventions. Preserve existing user work and do not commit a change until its relevant checks pass.

For strategist/executor work, run `.agentic/validate-contract-version.sh .`, then read `.agentic/protocol.md`, `.agentic/EXECUTOR.md`, and `.agentic/PROJECT_POLICY.md`. The installed protocol is version-pinned; repository policy overrides its defaults where more specific.
